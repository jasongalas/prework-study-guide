HTML

  • The head element contains information about the webpage. This is not visibile to the user.
  • The body element represents the visible content shown to the user.
  • The headers are numbered sequentially [i.e hi, h2, etc], and h1 should only appear once.
  • Images will have an src (source) that defines the location of the image and an alt (image description).
  • Links are anchored with the HREF containing the URL and a title to make it appear clean in a browser.

Git

  • git <— precedes every git statement
  • git checkout = moves working branch to a new branch (ex: git checkout "name of branch")
  • git -b = creates a new branch (ex: git checkout -b feature/starter code); "Checkout" moves working branch to the new branch we just created, feature/startercode is the name.
  • git status = checks what branch we’re currently working on. We always want to ensure we aren’t working on the main branch, but in our own branches that we’ll commit later.
  • git add -A = add command adds modifications in the current branch to staging area, -A means we want to add all changes
  • git commit -m "add starter code" = commits our branch; -m is the message, and the quotes containing the description of what’s been modified.
  • git pull origin main = pull <— receives a branch modification into the local environment. Origin <— indicates the source of the pull will be the GitHub Repo. "Main" <— Indicates the main branch.
  • git push origin feature/starter-code = push <— Create a new remote branch or add changes to an existing branch. Origin <— indicates the source of the pull will be the GitHub Repo. "feature/starter-code" <— is the branch we're committing.
